Module: protect

Middleware principal para proteger rutas verificando el Token JWT. Realiza las siguientes comprobaciones: 1. Verifica si el Token está presente en la cabecera 'Authorization' con el formato 'Bearer '. 2. Decodifica y verifica la firma del Token usando la clave secreta (JWT_SECRET). 3. Busca el usuario en la base de datos usando el ID decodificado para asegurar que el usuario existe. 4. Si el token es válido y el usuario existe, adjunta el objeto del usuario a 'req.user' y llama a next(). [Image of JWT verification flow]
Parameters:
Name Type Description
req object Objeto de petición HTTP, se espera 'req.headers.authorization'.
res object Objeto de respuesta HTTP.
next function Función para pasar al siguiente controlador o middleware.
